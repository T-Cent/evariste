Intro to LaTeX\LaTeXLATE​X

With a doc in markdown for the memes.

What is it?

It is a markup language, like HTML, we use

<h1>Heading</h1>

To define how hṭeading should appear, we define how and where text should appear and LaTeX\LaTeXLATE​X does the rest.

Every academic/research paper you see is written in LaTeX\LaTeXLATE​X, if it’s not, you should stay away from it. It’s used to create good looking documents with minimal effort. For example to get, ∫0πsin⁡(x)dx,\int_0^\pi\sin(x)dx,∫0π​sin(x)dx, we just need to type

\[\int_0^\pi\sin(x)dx\]

(Markdown with KaTeX\KaTeXKATE​X can also render LaTeX\LaTeXLATE​X, which is why you can see the nice equations here as well. Note that this does not make LaTeX\LaTeXLATE​X obsolete, as markdown is shit.)

But to quickly mention, for inline math, wrap the equation in $, like $x=2$, which generates x=2x=2x=2 or $$, like $$x=2$$ to get x=2.x=2.x=2.

Getting started with it

You have two options.

  1. Use an online IDE, such as Overleaf
  2. Download and run it locally

On Overleaf

Visit Overleaf and start a new blank project. We can directly start writing on the main.tex file, and see it’s final output on the left PDF after compilation.

Locally

To run LaTeX\LaTeXLATE​X locally, we need to download it. The simplest way is to use Tex Live. Follow the install instructions as per your operating system. (Note that the entire TeX\TeXTE​X package costs about 9 gigs, you can choose a smaller subset.)

After installation, you could get any TeX\TeXTE​X editor, visit Wikipedia’s list of TeX editors to choose one, I’d recommend TeXstudio.

If you’d like to us Visual Studio Code (which I prefer), get the LaTeX Workshop extension. You could also configure neovim with the VimTeX extension if you like.

Now, create any file ending with .tex, and use the pdflatex executable/binary to compile this TeX\TeXTE​X file into a PDF. The final command would be

pdflatex main.tex

Assuming you named your file main.tex. Note that every editor does this itself, so you need not worry about these commands.

LaTeX\LaTeXLATE​X

Let’s finally talk about it. The minimum you need to write to get started with any document is

\documentclass[options]{article}
\begin{document}
<Everything goes inside here>
\end{document}

Where the documentclass defines how our document looks like. article is the most common, but you could also have book, report, and more (even custom stuff). The options defines some parameters such as font size (default is 10pt), paper size (letterpaper/a4paper), landscape and more.

The next most important thing is setting up your document title, authors, and date.

\documentclass{article}

\title{Intro to \LaTeX}
\author{Taraash}
\date{\today} %which is default, so could be omitted.

\begin{document}
\maketitle

Welcome to some \LaTeX-ed!

\end{document}

Use the maketitle command inside the document environment, and define your title, author, and date as shown above. Use the percent symbol (%) for comments.

Document structure

We have \section{}, \subsection{} and \subsubsection{} to organize. Note that there are also starred version of these commands, like \section*{}, these do not number whatever section you have. \\ is the line break. \newpage moves everything below to a new page.

Use the \tableofcontents to… get a table of contents. Starred ((sub)sub)sections are hidden.

Math mode

Probably the most useful thing TeX\TeXTE​X is known for, its ability to handle extremely complex mathematical equations beautifully. Consider for example ∫0∫01e−x2ex2dx.\int_0^{\int_0^1e^{-x^2}}e^{x^2}dx.∫0∫01​e−x2​ex2dx. You would have incredible trouble getting something like this any other package, but in here, we just need \int_0^{\int_0^1e^{-x^2}}e^{x^2}dx to do it!

I’ve already described inline (use $math$ or \(math\)) and display math mode (use \[math\] or the equation environment). By an environment we mean

\begin{equation}
<stuff here>
\end{equation}

Hence document is also an environment, stuff inside which get’s displayed in the compiled PDF. (Much like the body tag in HTML, everything before the \begin{document}, the preamble is the head tag.)

  1. Subscripts: a_1 = a1a_1a1​, a_12 = a12a_12a1​2, a_{12} = a12a_{12}a12​
  2. Superscripts: x^2 = x2x^2x2, x^2y = x2yx^2yx2y, x^{2y} = x2yx^{2y}x2y
  3. Fractions: \frac{x}{y} = xy\frac{x}{y}yx​
  4. Square roots: \sqrt[n]{x} = xn\sqrt[n]{x}nx​
  5. Sum: \sum_{k=0}^{\infty}a_k = ∑k=0∞ak\sum_{k=0}^{\infty}a_k∑k=0∞​ak​
  6. Product: \prod_{k=0}^{\infty}a_k = ∏k=0∞ak\prod_{k=0}^{\infty}a_k∏k=0∞​ak​
  7. Integral: \int a_k = ∫ak\int a_k∫ak​

Align

\begin{align*}
x + 2 &= 7 \\
x &= 5
\end{align*}

x+2=7x=5 \begin{align*} x + 2 &= 7 \\ x &= 5 \end{align*} x+2x​=7=5​

The & are aligned vertically, here, keeping them before the equals to sign ensures the equals to sign are aligned vertically. Here, starring the environments gets rid of the numbering done by the unstarred environment.

x+2=7x=5 \begin{align} x + 2 &= 7 \\ x &= 5 \end{align} x+2x​=7=5​​

Quote

\begin{quote}
We are made of star-stuff.
\hfill \textit{---Carl Sagan}
\end{quote}

To get quotes, you need `hello’ (‘hello’\text{`hello'}‘hello’) and ``hello’’ for double (“hello"\text{``hello"}“hello"). DO NOT USE "baaad" ("baaad"\text{"baaad"}"baaad")!

Spacing

  • hspace{10pt}: Adds a horizontal space of 10pt
  • vspace{10pt}: Vertical space of 10pt
  • vfill: Adds space to vertically fill the page
  • hfill: Adds space to horizontally fill the page
  • newpage: Creates a new page
  • \\[10pt]: Adds a new line and a vertical space of 10pt
